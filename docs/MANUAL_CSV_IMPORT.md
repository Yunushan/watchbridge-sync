# Manual CSV Import

For services without a documented API, WatchBridge accepts a user-downloaded CSV together with an explicit column map. This preserves a compliant workflow: no scraping, password collection, or browser automation is needed.

Map a media export's title column and any available year, external IDs, rating, watched date, watchlist date, review text, review date, and spoiler flag. A social-only export may instead map `followingUsername` and/or `followerUsername`, with optional display name, absolute HTTPS profile URL, and follow date. Provide the rating scale used by the service whenever a rating column is mapped. The importer yields all six canonical arrays. Review spoiler values accept explicit true/false, 1/0, yes/no, or y/n forms and fail the review row closed when malformed. Social usernames reject surrounding whitespace/control characters, malformed dates or profile URLs fail that relationship closed, and duplicates are removed within each direction.

This workflow is intended for the 13 `manual-mapping` entries in the runtime registry: TV Time, Metacritic, Reelgood, Serializd, AllMovie, Criticker, FilmAffinity, Flickchart, Taste.io, MUBI, Common Sense Media, Douban Movie, and Kinopoisk. It does not imply that a service offers an export or that WatchBridge has a direct read/write path. Rotten Tomatoes, JustWatch, and AniList are restricted and are not part of this workflow.

The web app includes a **Manual CSV import** panel for this workflow. Paste a user-downloaded CSV, select its source service, adjust the explicit column map, preview the canonical records, then download the resulting local JSON backup. Following/follower usernames remain scoped to that source service; this workflow does not infer that the same handle belongs to the same person elsewhere.
